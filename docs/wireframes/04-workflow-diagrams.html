User Workflow Diagrams

Breadfast Pricing Intelligence — Key user journeys with decision points and error paths

A

Find and Fix Overpriced Products

Target: Complete in under 60 seconds
Commercial Analyst
User Action
Log in → Land on Commercial Dashboard
KPI strip loads immediately; treemap renders with latest data
System
Display KPI strip — Avg Blended PI: 1.0724
User glances at PI, sees it's close to parity in some areas
User Action
Scan treemap → Notice red cells (PI < 1.0)
Wipes (0.82) and Baby Food (0.89) stand out as problem areas
User Action
Click "Wipes" cell in treemap
Triggers filter cascade across entire page
System Response
All components filter to Wipes subcategory
Ranking table highlights row; Product table shows 142 products; KPIs update
User Action
Sort Product table by Sale PI ascending
Worst offenders (most overpriced) float to top
User Action
Find product where PI < 1.0 → Click price cell
Cell transforms to editable input with current value pre-filled
System
Inline editor appears with validation
Input accepts numbers only; shows Talabat price for reference
User Action
Enter new price → Press Enter or Tab to next
Decision: API Response
Success Path
Toast: "Price updated: EGP 42 → EGP 36"
Cell flashes green; PI recalculates inline; KPIs update
Error Path
Toast: "Sync failed — Retry?"
Cell shows error state (red border); user can retry or undo
User Action
Continue editing prices or click "Export CSV"
CSV includes all filtered products with old/new prices and PI
Complete
Session complete — all edits synced to backend
Average time: 45 seconds for identify + fix cycle

Design Notes

  • Tab key advances to next editable price cell for keyboard-driven workflows
  • Esc cancels edit and restores previous value
  • Ctrl+Z provides undo within session (up to 50 actions)
  • Bulk select with Shift+Click allows multi-product price adjustments
  • Auto-save with 500ms debounce prevents accidental double-submissions
B

Review AI-Suggested Matches

Process 87 pending matches in a single session
Master Data Specialist
User Action
Navigate to Master Data tab
System
Action Summary loads — "87 Review AI Match" highlighted
KPI card pulses subtly to draw attention
User Action
Click "Review AI Match" KPI card → Filter activates
Worklist and AI Panel filter to show only AI match candidates
System
AI Match Review Panel shows scrollable cards
Sorted by similarity score descending; highest confidence first
User Action
Review first card: BF product ↔ Talabat suggestion
Card shows both product names, prices, and 92% similarity bar
Decision: Is this a correct match?
Accept
Click "Accept" → Product status → Complete
Card fades out with green flash; counter: "86 remaining"
Reject
Click "Reject" → Product status → Needs Mapping
Card fades out; product moves to manual mapping queue
User Action
Continue reviewing remaining cards (keyboard: A=Accept, R=Reject)
User Action
Check Staleness Heatmap for next priority area
Dark cells in "21+ days" column indicate where to focus manual updates
Complete
Session complete — coverage % updated in real-time

Design Notes

  • Keyboard shortcuts: A to accept, R to reject, S to skip/defer
  • Cards pre-load next 5 in background for smooth scrolling
  • Undo last action available via Ctrl+Z or undo button in toast
  • Progress bar shows "12 of 87 reviewed" at top of panel
  • Low-confidence matches (below 70%) show warning amber border
C

Weekly Competitive Check

30-second executive overview
VP Commercial / Executive
User Action
Log in → Auto-lands on Executive Dashboard
Executive role auto-routes to executive view (configurable)
System — Instant (<2s)
PI Gauge shows 1.07 — "Breadfast is 7% cheaper overall"
Green color = positive position; trend arrow shows WoW improvement
User Scan — 5 seconds
Glance at Coverage (47.5%), Used Products (2,087), Actions (1,438)
User Action
Check "Most Expensive" column → See Baby Care at 0.92
Red indicators immediately flag problem areas
User Action
Click "Baby Care" row → Navigate to Commercial Dashboard
Cross-view navigation: auto-applies Main Category = Baby Care filter
System
Commercial Dashboard opens filtered to Baby Care
Treemap shows Diapers, Baby Food, Wipes subcategories
User Action
Review specific products → Forward to team
Export CSV or share dashboard link with pre-applied filters
Complete
Total time: ~30 seconds for overview; ~2 min for drill-down

Design Notes

  • Executive view has no filter bar — designed for consumption, not manipulation
  • Clicking any category/subcategory navigates to Commercial with filters pre-applied
  • Shareable URLs encode filter state for team collaboration
  • Weekly email digest option sends this view as PDF every Monday
D

First-Time Login & Data Loading

Onboarding and initial data sync
All Users
User Action
Open app URL → See login screen
Branded login page with Breadfast logo and "Sign in with Google" button
User Action
Click "Sign in with Google" → Google OAuth flow
Restricted to @breadfast.com domain
Decision: Valid @breadfast.com email?
Valid
Authentication succeeds → Redirect to app
Invalid
Error: "Please use your @breadfast.com account"
Redirect back to login with error message
System — Loading Phase 1 (3-5s)
Loading BigQuery data → Progress bar: "Loading product catalog..."
Skeleton loaders shown for all dashboard components
System — Loading Phase 2 (5-10s)
Fetching live catalog prices → "Enriching with live prices..."
Progress: "2,450 of 9,247 products enriched" with animated bar
System — Loading Phase 3 (1-2s)
Computing PI metrics → "Calculating price indices..."
Ready
Dashboard fully loaded — all data ready
Skeleton loaders dissolve into actual content with subtle fade-in

Design Notes

  • Loading screen shows Breadfast branding with animated progress
  • Subsequent logins skip BigQuery phase (cached) — only live prices refresh
  • If enrichment fails, show dashboard with stale data + warning banner
  • First-time users see optional 30-second feature tour tooltip sequence
  • Role-based routing: Executives → Executive tab; Analysts → Commercial tab
